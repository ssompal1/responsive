A Fresh Look For the New York
Department of Building's Search Site

Redesigning for simplicity, usability, and accessibility for public good

Last year, I learned through a grassroots collective of Asian migrant workers about how they used the New York Department of Buildings Search site in order to find out if their workplaces had tickets from the city. However, workers described the website as difficult to use, preventing them from accessing the information.

What could a redesigned website that improved accessibility and usability look like?

INITIAL EVALUATION

The website had some design benefits but largely presented significant usability issues.

  • The one strong benefit was that the site was fairly efficientâ€”once users knew how to navigate the page, users input all information on one page
    • Related searches grouped together
    • Poor hierarchy may slow users down, see below
  • However, it was difficult to learn to use the first time
    • Little hierarchy: Small titles and similar sizing for subtitles and text
    • No instructions or guidance for first-time users
    • Unclear where user should start navigating the page, how much info to enter
    • ++: Dropdowns, buttons, and links did provide visual affordances so users know where to look
      • Link references followed standard blue and underline format
      • Buttons had background color and borders
  • It is also difficult to remember how to navigate the page once the user has already visited it
    • Few visual cues for users to rely on

The site also had significant accessibility issues. I used the WebAIM WAVE Tool to provide a summary of site accessibility.

  • The site is largely unnavigable with a screenreader
    • Most errors from WAVE associated with lack of adequate form labels and alternative text for the dropdown menus and input form boxes
    • Many alerts around use of HTML layout tables → Introduce reading and navigation order issues for screenreaders
  • Users might also benefit from larger text

What do these usability and accessibility issues mean for user experience?

  • Visually impaired users using screenreaders cannot use this site and perform the searches they want
  • Other users may be intimidated by the site, and not know where to begin
  • Users may be frustrated with or discouraged by the site, not obtaining the info they seek
    • Consistent with feedback from massage workers.

MOCKUP DESIGN

I began by creating a style guide consistent with the City of New York's government sites' color palette and fonts.

From there, I created three high-fidelity wireframes for three-different screen sizes: a large desktop, a tablet, and a phone. In this redesign, I aimed to keep the simplicity of NYC websites and reduce complexity by hiding additional functions behind specific user paths distinguished by buttons.

MOCKUP IMPLEMENTATION & REFLECTION

I've implemented the home page of the website in order to implement the mockups. The redesigned page offers several key impovements:

  • Usability
    • Improved hierarchy of text and visual cues:Titles are clear and buttons augmented by icons direct users attention to potential types of searches
    • Highlight instructions for users: Menu bar items for FAQ and Glossary are made larger and higher up for users to seek guidance
    • Clear user paths hide unnecessary complexity:Hiding search behind buttons reduces text density on page
  • Accessibility
    • Eliminate HTML layout tables
    • Increased text size
    • Reducing text density improves initial screen navigation on screen reader
    • Improved hierarchy with proper navigation labels and alternative text

    These changes ultimately improve user experience by making the interface less overwhelming and clearly indicating possible interaction paths and additional guidance for users.

For websites intended for public service information sharing, designing with usability and accesibility in mind is especially important. I learned that maintaining simplicity and creating clear visual cues provided an effective way to redesign this website.
